  1. Go to https://github.com/ and create a GitHub account.

    NOTE: Your username will be publicly viewable, and will also be included in the URL for your website, so make sure to pick something you're comfortable sharing with the world!

  2. Login to your GitHub account. From your home screen, click the green "Create repository" button.



  3. Name your new repository:your_username.github.io (replacing your_username with your actual username, of course)

    NOTE: This is very important to get right!



    Make sure it is public, and tell GitHub to create a README file as well. Then click the green "Create repository" button.

  4. Now that you've created the repository, we're going to confirm that Github Pages is working correctly.  You should be on the repository page after creating it, so click on the "Settings" tab in the navigation near the top.



  5. From there, click the "Pages" link in the left sidenav.



  6. You should see the link to your brand new website in the green box.



    Congrats!  You've created your first website (it's just very empty at the moment).

  1. If you haven't already, download and install GitHub Desktop from the following site: https://desktop.github.com/

  2. After installation, log in using your new GitHub account. It will take you briefly back to the browser to confirm your identify; click "Allow" when prompted. You should see some pages in the app like the following:

    Here's the page you will see when it redirects you to Github to authenticate yourself.



    On the Configure Git, select the "Use my Github account name and email address".


  3. After you've successfully logged in to GitHub Desktop, we are going to "clone" your remote repository to your local machine.

    What this means is, we are going to pull down and create a copy of the repository from Github.com (the remote location) to your Desktop (the local location).

  4. Create a new folder on your desktop called "GitHub".


  5. In GitHub Desktop, click the "Clone a Repository from the Internet" button.



  6. Select the Repository we just created earlier on Github.com, and select the folder on your Desktop you just created (you may need to add your_username.github.io to the file path afterwards if it doesn't auto-populate it; see the screenshot below).



  7. After clicking "Clone", Github Desktop will download your repository from the web.  After the download, you should see a new folder named after your repository, with a readme file inside it. This means you successfully cloned your repository!

    Make sure you pay attention to the local path where your project is cloned to so you can always find it on your computer!

  1. If you haven't already, download and install the Visual Studio Code (found here: https://code.visualstudio.com/)
  2. Reopen Github Desktop (Your cloned repository should be the current) and Click "Open in Visual Studio" in the middle section
  3. Now we will install a few extensions to be able to view your live code within your code editor

    First, navigate to the left side panel in Visual Studio Code and click the icon with the four boxes.


    In the Extensions tab, click in the search bar at the top. Then in the search bar, type "live preview". The extension Live Preview by Microsfoft should be one of the first in the list. Click the Install button to install the extension


    This extension will allow you to preview your HTML/CSS/Javascript code within Visual Studio.

    In the Extensions tab, click in the search bar at the top. Then in the search bar, type "live server". The extension Live Server by Ritwick Dey should be one of the first in the list. Click the Install button to install the extension


    This extension will allow you to preview your HTML/CSS/Javascript code within Visual Studio.

  4. Now find your code skeleton folder that we created earlier and drop the entire folder into your repository in Visual Studio.

    Code Skeleton in Finder

    Ask to Copy Folder



    Code Skeleton in Finder



    NOTE: index has a special meaning in most web servers; if you access a folder without specifying a file, the index page will get loaded. What this means for you, is that when you visit your web site URL, index.html will become the home page.

  5. Now that we have setup the technology we will be utilizing, we're going to create a basic "Hello World!" homepage!

    Make sure to save your changes before moving on to the next step; if you have a little blue circle next to the file name, that means it has unsaved changes in it.

  6. Follow instructions on Canvas to complete "Hello World" Assignment with the new tags you've been introduced to.

  7. Now, jump back to Github Desktop. In the app, you should see the new index.html file you added, along with its contents.



    We're going to save a copy of your changes in something called a "commit". A commit is basically a snapshot of the changes you made, which gets added to the repository's source control. This is useful for many reasons; for example, if you make a breaking change to your website, you can revert back to a previous commit (aka snapshot) to make it work again while you try to figure out what went wrong.

    Add a commit name (required) and description (optional) in the textboxes in the bottom left of the app. Then click the "Commit to main" button to create a commit of your changes.



    WARNING: Github cannot handle super large files, and adding files greater than 20 MB could cause Github Desktop to choke; it is highly recommended to use compressed files on the web anyway, so if you have high resolution image or video files you'd like to use, talk with your instructor first.  You will likely make your school life a lot harder otherwise...

  8. Next, we are going to do something called a "push". A push is an action that, well, pushes your local commits to the remote server on github.com.

    There are two places you start a push in Github Desktop, both circled in the screenshot below. (The term origin refers to the repository on the remote server (github.com) that you cloned your local repository from.)

    Note that each location has an indicator of how many commits you are going to push. That is because it's possible to make multiple commits locally before pushing them.



    Go ahead and click either option to push your changes to your website.

  9. If the push was successful, the top option should change to "Fetch origin".



    For reference:

    Fetching     is a command to tell Github Desktop to communicate with the remote server repository to see if anything changed since the last time we fetched.

    If there were any changes (which could happen if you were writing code on multiple computers or with a team of developers who were pushing their own commits), that button would change to "Pull origin", and indicate how many commits that you're missing.

    Pulling is the process of downloading/syncing changes from the remote server repository with your own local repository.

  10. Now return to the GitHub website, and go into the repository you made earlier with github.io at the end of it. Once you navigate to the main repository code screen, you should see your HelloWorld folder that you created in the middle section.

    NOTE: Github pages often times takes 5-10 minutes to actually render out your commit, so you may have to wait and then check to see if your site is live

    Now you can navigate to the sidebar and click on github pages under "Environments".



    From there you will find the deployment page (the most recent deployment will usually be at the top). Click on the deployment and it will open your "homepage" of sorts. In order to find your Hello World site (a project nested in your main repository), you must add the project folder name to the end of the URL

    For example:
    https://gdwd-kc.github.io/kctobias.github.io/HelloWorld/

    Don't forget the last forward slash!

    Within a couple minutes of pushing your code, the url should update to display your brand new "Hello World" homepage!

    Congratulations! You just built your first GitHub Pages site.
